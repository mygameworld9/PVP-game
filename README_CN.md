# 角色阵容游戏 (Character Roster Game)

一个具有多样化角色阵容、独特属性和状态管理的2D游戏，支持中英文双语界面。

## 🎮 游戏特色

### 角色系统
- **多样化角色阵容**: 战士、法师、射手三种独特角色
- **独特角色属性**: 每个角色都有独特的技能、移动速度、生命值、攻击风格和伤害输出
- **角色状态管理**: 使用有限状态机(FSM)处理角色的各种状态

### 角色详情

#### 🗡️ 战士 (Warrior)
- **生命值**: 150 HP
- **移动速度**: 180
- **攻击伤害**: 35
- **攻击风格**: 近战
- **技能**:
  - 基础攻击: 快速近战攻击
  - 盾击: 高伤害近战攻击，带击退效果
  - 战吼: 提升攻击伤害和移动速度

#### 🔮 法师 (Mage)
- **生命值**: 100 HP
- **移动速度**: 150
- **攻击伤害**: 25
- **攻击风格**: 远程
- **技能**:
  - 基础攻击: 快速魔法攻击
  - 火球术: 高伤害远程攻击
  - 传送: 瞬间移动到鼠标位置

#### 🏹 射手 (Archer)
- **生命值**: 120 HP
- **移动速度**: 200
- **攻击伤害**: 30
- **攻击风格**: 远程
- **技能**:
  - 基础攻击: 快速箭矢攻击
  - 多重射击: 发射多支箭矢
  - 冲刺: 快速向前移动

## 🎯 控制方式

### 移动控制
- **WASD**: 角色移动
- **Shift**: 跑步模式

### 攻击控制
- **空格键**: 基础攻击
- **左键点击**: 攻击1 (每个角色都有独特的攻击1)
- **右键点击**: 攻击2 (每个角色都有独特的攻击2)

### 技能控制
- **1键**: 技能1
- **2键**: 技能2
- **3键**: 技能3

## 🌍 语言支持

### 双语界面
- **英文模式**: 完整的英文界面
- **中文模式**: 完整的中文界面
- **实时切换**: 点击右上角的语言按钮即可切换

### 界面翻译
- **游戏标题**: Character Roster Game / 角色阵容游戏
- **角色名称**: Warrior/战士, Mage/法师, Archer/射手
- **状态信息**: Health/生命值, State/状态
- **控制说明**: 所有控制按键都有中英文说明

## 🚀 如何运行

### 系统要求
- **操作系统**: Windows 10/11
- **Godot版本**: 4.4.1 或更高版本
- **内存**: 至少 4GB RAM
- **显卡**: 支持 OpenGL 3.3 或 Vulkan

### 安装步骤

1. **下载Godot引擎**
   ```
   访问 https://godotengine.org/download
   下载 Godot 4.4.1 或更高版本
   ```

2. **打开项目**
   ```
   启动 Godot 引擎
   点击 "导入" 按钮
   选择 character_roster_game 文件夹
   点击 "导入并编辑"
   ```

3. **运行游戏**
   ```
   在编辑器中按 F5 键
   或点击 "播放" 按钮
   ```

### 命令行运行
```bash
# 进入项目目录
cd character_roster_game

# 使用Godot运行项目
& "D:\Godot\Godot_v4.4.1-stable_win64.exe" --path "D:\game\character_roster_game"
```

## 🎨 技术特性

### 动画系统
- **AnimatedSprite2D**: 使用精灵帧动画系统
- **状态动画**: 空闲、行走、跑步、攻击、受伤、击败
- **角色动画**: 每个角色都有独特的动画效果

### 状态管理
- **有限状态机**: 管理角色的各种状态
- **状态转换**: 平滑的状态切换
- **状态保护**: 防止在攻击或受伤时执行其他动作

### 输入系统
- **键盘输入**: WASD移动，数字键技能
- **鼠标输入**: 左右键攻击
- **输入映射**: 可自定义的控制映射

### 语言系统
- **自动加载**: LanguageManager作为全局单例
- **动态翻译**: 实时语言切换
- **信号系统**: 语言变化时自动更新UI

## 🐛 已知问题

### 动画错误
```
ERROR: There is no animation with name 'attack'.
```
**解决方案**: 确保角色场景中的AnimatedSprite2D包含正确的动画名称

### UID冲突
```
WARNING: UID duplicate detected
```
**解决方案**: 每个场景文件都有唯一的UID标识符
